## 详细索引

### 📋 project-summary

- **文件**：[project-summary.md](./project-summary.md)
- **功能**：高质量项目经历生成，内置写作方法论、差异化策略、避坑清单和岗位适配策略，8-12行内产出满分简历项目经历
- **适用场景**：
  - 需要为简历准备高质量的项目经历描述
  - 需要针对不同岗位级别（初/中/高级/架构师/管理）调整内容侧重
  - 需要量化项目成果并体现技术深度和决策能力
- **输入要求**：一个可分析的项目代码库（建议同时提供目标岗位级别）
- **输出格式**：精炼的简历项目经历文本（项目头信息 + 3条核心职责 + 2条技术亮点，每条一句话，总计8-12行）
- **关键词**：`简历`, `项目总结`, `量化指标`, `差异化`, `岗位适配`, `写作方法论`
- **最后更新**：2026-04-09

## AI 使用指南

### 🔍 如何匹配 Skill

1. **关键词匹配**：根据用户需求中的关键词，在上方「快速导航」表格的「关键词」列中查找匹配项
2. **场景匹配**：在「详细索引」中查看每个 Skill 的「适用场景」，选择最符合当前任务的 Skill
3. **组合使用**：如果任务涉及多个方面，可以组合使用多个 Skill

### 📖 使用流程

```
1. 查阅本索引 → 匹配合适的 Skill
2. 读取对应 Skill 文件 → 了解执行步骤和输出模板
3. 按照 Skill 定义的流程执行任务
4. 按照 Skill 定义的模板输出结果
```

## 🔧 索引自动维护规则

> **强制要求**：AI 在对 `skills/` 目录进行任何文件变更时，**必须同步更新本索引文件**。

### 维护触发条件

以下操作**必须**触发索引更新：

1. **新增 Skill 文件**：在 `skills/` 目录下创建新的 `.md` 文件
2. **删除 Skill 文件**：从 `skills/` 目录下删除 `.md` 文件
3. **修改 Skill 文件**：修改了 Skill 的功能描述、适用场景或输出格式等核心内容
4. **重命名 Skill 文件**：更改了 Skill 文件的文件名

### 维护操作清单

当触发条件满足时，AI 需要执行以下更新操作：

#### 新增 Skill 时
- [ ] 在「快速导航」表格中添加新行
- [ ] 在「详细索引」中添加新的 Skill 详情块（参照已有格式）
- [ ] 更新「索引元信息」中的 `Skill 总数` 和 `最后更新` 日期
- [ ] 在「索引更新记录」中添加变更记录

#### 删除 Skill 时
- [ ] 从「快速导航」表格中移除对应行
- [ ] 从「详细索引」中移除对应的 Skill 详情块
- [ ] 更新「索引元信息」中的 `Skill 总数` 和 `最后更新` 日期
- [ ] 在「索引更新记录」中添加变更记录

#### 修改 Skill 时
- [ ] 更新「快速导航」表格中对应行的描述和关键词（如有变化）
- [ ] 更新「详细索引」中对应 Skill 的详情（如有变化）
- [ ] 更新对应 Skill 详情块中的 `最后更新` 日期
- [ ] 更新「索引元信息」中的 `最后更新` 日期
- [ ] 在「索引更新记录」中添加变更记录
