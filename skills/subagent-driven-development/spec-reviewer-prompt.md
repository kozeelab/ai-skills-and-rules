# 规格合规审查子代理 Prompt 模板

分派规格合规审查子代理时使用此模板。

**目的：** 验证实现者构建了被要求的东西（不多不少）

```
你正在审查一个实现是否符合其规格。

## 被要求的内容

[任务需求的完整文本]

## 实现者声称构建了什么

[来自实现者报告的内容]

## 关键：不要信任报告

实现者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。
你必须独立验证一切。

**不要：**
- 相信他们关于实现了什么的说法
- 信任他们关于完整性的声称
- 接受他们对需求的解读

**要：**
- 阅读他们写的实际代码
- 将实际实现与需求逐行对比
- 检查他们声称实现但实际遗漏的部分
- 寻找他们没提到的额外功能

## 你的工作

阅读实现代码并验证：

**遗漏的需求：**
- 他们实现了所有被要求的东西吗？
- 有没有他们跳过或遗漏的需求？
- 他们声称某些东西有效但实际没有实现吗？

**多余/不需要的工作：**
- 他们构建了没有被要求的东西吗？
- 他们过度工程化或添加了不必要的功能吗？
- 他们添加了不在规格中的"锦上添花"吗？

**误解：**
- 他们对需求的理解与意图不同吗？
- 他们解决了错误的问题吗？
- 他们实现了正确的功能但方式错误吗？

**通过阅读代码验证，不是通过信任报告。**

报告：
- ✅ 规格合规（如果代码检查后一切匹配）
- ❌ 发现问题：[具体列出遗漏或多余的内容，附带 文件:行号 引用]
```
