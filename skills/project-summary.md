# 项目总结 Skill

## 功能说明

此Skill用于总结项目中的亮点和难点，生成一段适合直接复制到简历上的项目经历。

## 使用方法

1. 分析项目结构和代码
2. 识别项目的技术亮点和挑战
3. 生成结构化的项目经历描述
4. 输出适合简历的专业语言

## 执行步骤

1. **项目分析**
   - 扫描项目目录结构
   - 识别使用的技术栈和框架
   - 分析项目的核心功能和模块

2. **亮点提取**
   - 技术创新点
   - 性能优化措施
   - 架构设计优势
   - 解决的关键问题

3. **难点分析**
   - 技术挑战
   - 解决方案
   - 克服的困难

4. **简历输出**
   - 结构化的项目描述
   - 量化的成果指标
   - 专业的技术术语
   - 清晰的职责描述

## 量化指标示例

### 性能优化
- 响应时间减少：从X秒优化到Y秒，提升Z%
- 系统吞吐量：从每秒处理X请求提升到Y请求，增长Z%
- 内存使用：减少X%，节省YGB内存
- CPU利用率：降低X%，提升系统稳定性

### 业务成果
- 用户增长：用户数从X增长到Y，增长率Z%
- 收入提升：销售额增加X%，达到Y万元
- 成本节约：降低运营成本X%，节省Y万元
- 效率提升：工作流程时间减少X%，提高产能Y%

### 技术指标
- 代码质量：代码覆盖率从X%提升到Y%
- 部署频率：从每周X次部署提升到Y次
- 故障时间：系统宕机时间减少X%，可用性达到Y%
- 开发效率：开发周期从X天缩短到Y天，提升Z%

### 团队协作
- 团队规模：从X人扩展到Y人，团队效率提升Z%
- 文档完善：文档覆盖率从X%提升到Y%
- 知识共享：建立X个知识库，覆盖Y个技术领域
- 培训效果：团队技能认证通过率从X%提升到Y%
