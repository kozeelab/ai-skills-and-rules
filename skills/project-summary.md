# 项目总结 Skill

## 功能说明

此Skill用于总结项目中的亮点和难点，生成一段**精炼的**、适合直接复制到简历上的项目经历。

> **核心原则**：简历项目经历应简洁有力，单个项目经历控制在 **8-12行** 以内，每条描述 **一句话** 说清楚，突出量化成果。

## 使用方法

1. 分析项目结构和代码
2. 识别项目的技术亮点和挑战
3. 生成精炼的项目经历描述
4. 输出适合简历的专业语言

## 执行步骤

1. **项目分析**
   - 扫描项目目录结构
   - 识别使用的技术栈和框架
   - 分析项目的核心功能和模块

2. **亮点提取**（挑选最有价值的 2-3 个）
   - 技术创新点
   - 性能优化措施
   - 架构设计优势

3. **简历输出**
   - 精炼的项目描述（1句话）
   - 核心职责（2-3条，每条1句话）
   - 技术亮点（2条，每条1句话）
   - 所有描述必须包含量化指标

## 输出模板

```
**[项目名称]** | [核心技术栈] | [团队规模] | [项目周期]
[一句话描述项目的目标、功能和业务价值]

核心职责：
1. [动作 + 做了什么 + 量化成果]
2. [动作 + 做了什么 + 量化成果]
3. [动作 + 做了什么 + 量化成果]

技术亮点：
1. [挑战/背景 + 解决方案 + 量化效果]
2. [挑战/背景 + 解决方案 + 量化效果]
```

## 写作要求

| 要求 | 说明 |
|------|------|
| **篇幅控制** | 单个项目经历总计 8-12 行，不超过 300 字 |
| **一句话原则** | 每条职责/亮点用一句话概括，不超过 50 字 |
| **量化优先** | 每条必须包含至少一个量化指标（百分比、倍数、具体数值） |
| **动词开头** | 职责描述以动词开头（设计、开发、优化、主导、搭建等） |
| **去除冗余** | 不写可迁移技能、行业经验等通用内容，只写本项目特有的 |

## 量化指标参考

| 类型 | 示例 |
|------|------|
| 性能 | 响应时间降低X%、吞吐量提升X倍、内存占用减少X% |
| 效率 | 开发周期缩短X%、部署频率提升X倍、人工操作减少X% |
| 质量 | 代码覆盖率提升至X%、故障率降低X%、可用性达X% |
| 业务 | 用户量增长X%、成本降低X万、转化率提升X% |
