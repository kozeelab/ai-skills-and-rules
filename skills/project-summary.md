# 项目总结 Skill

## 功能说明

此Skill用于总结项目中的亮点和难点，生成一段适合直接复制到简历上的项目经历。

## 使用方法

1. 分析项目结构和代码
2. 识别项目的技术亮点和挑战
3. 生成结构化的项目经历描述
4. 输出适合简历的专业语言

## 执行步骤

1. **项目分析**
   - 扫描项目目录结构
   - 识别使用的技术栈和框架
   - 分析项目的核心功能和模块

2. **亮点提取**
   - 技术创新点
   - 性能优化措施
   - 架构设计优势
   - 解决的关键问题

3. **难点分析**
   - 技术挑战
   - 解决方案
   - 克服的困难

4. **简历输出**
   - 结构化的项目描述
   - 量化的成果指标
   - 专业的技术术语
   - 清晰的职责描述

## 输出模板

```

项目描述：[简要描述项目的目标和功能]

个人职责：
1. [背景描述 + 负责工作 + 采取行动 + 取得成果]

3. [背景描述 + 负责工作 + 采取行动 + 取得成果]
4. [背景描述 + 负责工作 + 采取行动 + 取得成果]

技术亮点：
1. [背景描述 + 技术挑战 + 解决方案 + 实现效果]
2. [背景描述 + 技术挑战 + 解决方案 + 实现效果]

1. [面临挑战 + 解决思路 + 具体行动 + 最终结果]

3. [面临挑战 + 解决思路 + 具体行动 + 最终结果]
```

3. 代码变更需要手动提交和推送，容易出错且效率低，负责开发自动版本控制机制。编写了跨平台脚本，实现了代码变更的自动检测、提交和推送，实现了自动化的版本控制流程，减少了手动操作的错误率。
4. 团队成员需要为简历准备项目经历，缺乏标准化的总结方法，负责创建项目总结技能。设计了STAR法则模板，实现了项目亮点自动提取和简历内容生成，简化了简历内容的生成过程，提高了团队成员的职业竞争力。
