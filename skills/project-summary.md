# 项目总结 Skill

## 功能说明

此Skill用于总结项目中的亮点和难点，生成一段适合直接复制到简历上的项目经历。

## 使用方法

1. 分析项目结构和代码
2. 识别项目的技术亮点和挑战
3. 生成结构化的项目经历描述
4. 输出适合简历的专业语言

## 执行步骤

1. **项目分析**
   - 扫描项目目录结构
   - 识别使用的技术栈和框架
   - 分析项目的核心功能和模块

2. **亮点提取**
   - 技术创新点
   - 性能优化措施
   - 架构设计优势
   - 解决的关键问题

3. **难点分析**
   - 技术挑战
   - 解决方案
   - 克服的困难

4. **简历输出**
   - 结构化的项目描述
   - 量化的成果指标
   - 专业的技术术语
   - 清晰的职责描述

## 输出模板（STAR法则）

```
项目名称：[项目名称]

情境（Situation）：[描述项目背景和面临的挑战]

任务（Task）：[描述你的具体职责和目标]

行动（Action）：[描述你采取的具体行动和技术方案]

结果（Result）：[描述项目成果和量化的效益]
```

## 示例输出

```
项目名称：AI 技能与规则管理系统

情境（Situation）：随着AI开发的快速发展，缺乏标准化的开发规范和流程，导致开发效率低下，代码质量参差不齐，团队协作困难。

行动（Action）：
- 设计并搭建项目目录结构，建立清晰的规则分类体系，包括通用规则和语言特定规则
- 制定AI协作开发规范，包括项目扫描、任务执行流程等标准操作流程

- 创建项目总结技能，实现项目亮点自动提取和简历内容生成，提高团队成员的职业发展支持
- 设计详细的规则索引和使用指南，确保团队成员能够快速找到和应用相关规则

结果（Result）：
- 建立了一套完整的AI开发规范体系，提高了开发效率30%，代码质量显著提升
- 实现了自动化的版本控制流程，减少了80%的手动操作，降低了错误率
- 为团队提供了清晰的开发指导，新成员学习成本降低50%
- 创建的项目总结技能，简化了简历内容的生成过程，提高了团队成员的职业竞争力
```
