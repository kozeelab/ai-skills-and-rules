# 数据库操作规范

> **强制**：所有数据库操作必须遵守本文件中的规范

## 1. 数据库连接管理

### 1.1 连接池配置
- 使用连接池管理数据库连接
- 合理设置连接池大小
- 避免频繁创建和关闭连接

### 1.2 连接关闭
- 确保数据库连接在使用后正确关闭
- 使用 defer 确保连接释放

## 2. SQL 操作规范

### 2.1 参数化查询
- 所有数据库操作使用参数化查询
- 严禁 SQL 拼接
- 防止 SQL 注入攻击

### 2.2 事务管理
- 多表操作必须使用事务
- 确保事务的原子性
- 正确处理事务的提交和回滚

### 2.3 错误处理
- 数据库操作错误必须处理
- 错误信息应清晰明确
- 避免吞掉错误

## 3. ORM 使用规范

### 3.1 ORM 最佳实践
- 使用 ORM 的链式调用或查询构建器
- 合理使用预加载关联数据
- 避免 N+1 查询问题

### 3.2 模型定义
- 模型应包含必要的字段
- 使用适当的注解或配置定义字段属性
- 遵循命名规范

### 3.3 索引使用
- 为频繁查询的字段创建索引
- 合理使用复合索引
- 避免过多索引影响写入性能

## 4. 查询优化

### 4.1 分页查询
- 列表接口必须分页
- 限制每页最大数量
- 使用 `LIMIT` 和 `OFFSET` 优化查询

### 4.2 条件查询
- 使用 ORM 的条件构建器
- 避免复杂的 SQL 语句
- 合理使用条件组合

### 4.3 关联查询
- 合理使用预加载
- 避免深度关联查询
- 考虑使用连接查询优化性能

## 5. 数据完整性

### 5.1 约束设置
- 为必要字段设置 `not null` 约束
- 为唯一字段设置 `unique` 约束
- 使用外键约束确保数据一致性

### 5.2 数据验证
- 在业务层进行数据验证
- 确保数据的合法性
- 避免无效数据进入数据库

### 5.3 软删除
- 优先使用软删除功能
- 避免物理删除数据
- 使用专门的字段标记删除状态

## 6. 数据库迁移

### 6.1 迁移文件管理
- 使用迁移工具管理数据库结构变更
- 记录迁移历史
- 确保迁移的可重复性

### 6.2 迁移策略
- 开发环境：可以使用自动迁移
- 生产环境：应使用手动迁移脚本
- 迁移前备份数据

## 7. 性能监控

### 7.1 慢查询日志
- 开启慢查询日志
- 监控和分析慢查询
- 优化慢查询语句

### 7.2 连接监控
- 监控数据库连接池状态
- 避免连接泄漏
- 合理调整连接池参数

## 8. 安全规范

### 8.1 敏感数据处理
- 密码等敏感数据必须加密存储
- 禁止明文存储敏感信息
- 使用安全的加密算法

### 8.2 权限控制
- 数据库用户应遵循最小权限原则
- 不同环境使用不同的数据库用户
- 定期更新数据库密码

### 8.3 备份策略
- 定期备份数据库
- 制定灾难恢复计划
- 测试备份的可恢复性
