# 自动提交规则

## 规则说明

本文件定义了项目的自动提交规则，确保所有修改都能及时推送到远程仓库。

## 触发条件

- 任何代码或配置文件的修改
- 任何规则文件的更新
- 任何文档的变更

## 提交规范

- **提交信息格式**：每次提交使用不同的描述性信息
- **提交范围**：所有修改的文件
- **推送目标**：`origin main`

## 执行流程

1. 检测项目更改
2. 添加所有更改到暂存区
3. 生成描述性提交信息（每次不同）
4. 提交更改
5. 推送到远程仓库

## 注意事项

- 确保远程仓库连接正常
- 确保工作目录干净，避免冲突
- 推送前验证更改内容
- 每次提交使用不同的描述性信息

## 执行命令

在 Windows 环境中，执行以下命令：
```powershell
git add .; git commit -m "[描述性提交信息]"; git push origin main
```

在 Linux 或 macOS 环境中，执行以下命令：
```bash
git add . && git commit -m "[描述性提交信息]" && git push origin main
```

## 提交信息示例

- `chore: 更新规则和技能文档`
- `feat: 添加新项目总结功能`
- `fix: 解决脚本执行问题`
- `docs: 改进自动提交规则文档`
- `refactor: 优化代码结构`
- `perf: 增强脚本性能`
- `test: 为新功能添加测试用例`
- `style: 修复格式问题`
